Level00 - не храни зашифрованные шифром цезаря пароли в файле.

1. Заходим в дирректорию пользователя level00, пароль level00;

2. В дирректории нет файлов. 

3. Пробуем проверить список паролей в /etc/passwd. Среди них нет паролей для flag00, 
    но отметим для себя любопытную структуру для flag01;

4. Попробуем найти файлы, созданные пользователем flag00;

5. Команда find / -user flag00 возвращает излишне большой список файлов, к большинству 
    из которых у нас нет доступа. Сократим результат перенаправлением таких файлов в другой поток find / -user flag00 2> /dev/null.

6. Результат выборки содержит 2 файла: /usr/sbin/john и /rofs/usr/sbin/john,
    оба содержат одинаковые записи - cdiiddwpgswtgt. Строка текста. Предполагаем что это наш пароль,
    вводим его для пользователя flag00 - su flag00 + passwd.

7. Пароль не верен. Сделаем еще одно предположение - это шифр. Для начала ппопробуем шифр Цезаря.
    Программа-дешифровщик 00_decipher.py. Получаем список паролей-кандидатов. Наиболее многообещующий из них - nottoohardhere.

8. su flag00 + nottoohardhere -> Успех! Вводим команду gettoken. Получаем значение x24ti5gi3x0ol2eh4esiuxias. Переходим к уровню 01.
