Для того, чтобы порешать с пацанами последний уровень, требуется дизассемблировать /bin/getflag. Для этого по классике
objdump -M intel -C -d -s /bin/getflag

Во втором окне параллельно запускаем дебаггер.
Наша задача пройти через проверку от реверса + через проверку uid.

break *0x804898e # ставим брейкпойнт на месте проверки на реверс
run
set $eax = 0     # убеждаем программу, что все ок
break *0x8048b02 # ставим брейкпойнт на первую инструкцию после получения uid
continue
set &eax = 3014  # убеждаем программу, что мы flag14
continue

========
Вывод: не храни пароли (даже зашифрованные) в файлах, к которым потенциальный злоумышленник имеет доступ (в данном случае /bin/getflag). Уязвимости можно было бы избежать, если бы пароли хранились в файле с ограниченными правами доступа.
